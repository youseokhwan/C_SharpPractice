1. Node.js란
- Node.js : Javascript Runtime
* Runtime : 특정 언어로 만든 프로그램을 실행할 수 있는 환경
- 이벤트 기반, 논블로킹 I/O 모델을 사용해 가볍고 효율적임
- 꼭 서버로만 사용되는 것은 아님
- 브라우저 외에 환경에서 Javascript를 실행하기 위한 시도

2. 이벤트 기반
- 이벤트가 발생할 때 미리 지정해둔 작업을 수행
- 이벤트 리스너에 콜백 함수를 등록
- 여러 이벤트가 동시에 발생하면 이벤트 루프를 통해 판단

3. 논블로킹 방식
- 논블로킹 : 이전 작업이 완료될 때까지 기다리지 않고 다음 작업을 수행
- 동기/비동기 개념과 유사
- 기존에 setTimeout(콜백, 0) 메소드를 통해 논블로킹으로 만들었으나, 노드에서는 setImmediate(콜백)을 사용함
* 이때 setImmediate()가 setTimeout(0)보다 무조건 먼저 실행됨을 보장할 수 없으므로, setTimeout(0)은 사용하지 않는다.

4. 싱글스레드
- 자바스크립트 언어의 특성상 노드 또한 싱글스레드 방식을 채택
- 멀티스레드 대신 멀티프로세싱으로 구현

5. 노드의 장단점
- 멀티 스레드 방식에 비해 자원을 적게 사용
- I/O 작업이 많은 서버로 적합 / CPU 작업이 많은 서버로는 부적합
- 멀티 스레드보다 쉽지만, 스레드가 문제가 생기면 모든 작업 중단
- 웹 서버가 내장되어 좋지만, 규모가 크면 관리가 더 어려움
- 자바스크립트를 사용해 JSON과 호환성이 좋으나 성능이 낮음
